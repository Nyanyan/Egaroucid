# WTHORテストデータにおける着手一致率

関連issue: #613

更新日: 2026-07-20

## `hint 3`を使っていた旧結果の無効化

旧実装は、上位3手一致率を測るための候補数と、Egaroucidの方策を作るために必要な候補数を混同し、Consoleへ`hint 3`を要求していた。
その結果、取得した3手以外の合法手へ確率0を割り当てた状態でPolicy Networkとの幾何ブレンドを計算していた。
特に`0 < α < 1`では、本来選ばれ得る合法手がブレンド後も必ず確率0になるため、正しいブレンド方策ではない。

このため、`hint3`を名前に含む旧出力と、それを使ったブレンド方策の着手一致率は正式な実験結果として使用しない。
旧100,000局面評価を入力にした`../50_rating_vs_human_match/`のブレンド系列も、全合法手を使った再評価が完了するまで無効とする。
`../40_test_strength/`の強さ評価は従来から全合法手を取得する`hint 100`を使っていたため、この不具合の影響を受けない。

## 修正後の方策生成

修正後は次の条件をすべて満たさなければ、Egaroucidの方策を生成せずエラーにする。

- Consoleへ`hint 64`を要求する。Console側で実際の合法手数まで切り詰められるため、全合法手が返る。
- パースした手の集合が、盤面から計算した合法手の集合と完全に一致することを検証する。
- 新規出力だけでなく、メモリキャッシュとSQLiteキャッシュから読み込んだ出力も同じように検証する。
- キャッシュキーへ`hint`候補数を含め、旧形式のキャッシュと分離する。
- シャードの完了判定とマージでは、`console_hint_count=64`かつ`egaroucid_policy_support=all_legal_moves`を必須とする。

ブレンド方策は次の式で計算する。

```text
ブレンド方策 = (Policy Networkの方策)^α × (Egaroucidの方策)^(1 - α)
```

着手一致率は、手番側と相手側の石配置をそれぞれ不変に保つ、正方形盤面の回転・鏡映による8通りの変換を考慮する。
WTHORの実着手からそのような変換で移る合法手のいずれかが上位N手に入れば、一致と数える。

## 修正版の1,000局面検証

全合法手を保存した既存キャッシュ900状態について合法手集合との一致を監査し、900状態すべてで一致した。
さらに修正後の評価器へこのキャッシュを指定して同じ1,000局面を再評価し、キャッシュ読込1,000件すべてが新しい完全性検査を通過した。
検証出力は`output/random_wthor_test_n1000_seed613_level21_jobs4_threads8_hint64_all_legal_validated/`へ保存した。

## 実行

全合法手を新規に探索して再評価する場合は、次のコマンドを使う。

```powershell
python src/tools/policy_network_human_like_ai/20_test_with_wthor/run_random_wthor_blend_experiment.py 1000 `
  --data-split test `
  --split-seed 613 `
  --sample-seed 613 `
  --jobs 4
```

既定の出力先名には`jobs`、Consoleプロセスごとのスレッド数、`hint64`が含まれる。
異なる条件で作成したキャッシュや旧`hint3`出力は再利用しない。
