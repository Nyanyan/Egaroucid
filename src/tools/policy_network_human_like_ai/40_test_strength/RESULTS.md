関連issue: #613

## ブレンド方法

合法手上で正規化したPolicy Networkの方策とEgaroucid for Consoleの方策を、次の式でブレンドした。

```text
ブレンド方策 = (Policy Networkの方策)^α × (Egaroucid for Consoleの方策)^(1 - α)
```

α=0.0はEgaroucid for Consoleの方策のみ、α=1.0はPolicy Networkの方策のみを使う。Egaroucid側の方策生成レベルは21である。

## 対戦セットの定義

1対戦セットでは、1つのXOT局面から同じ2参加者が色を入れ替えて2局を行う。2局における同一参加者視点の石差を平均し、その符号で対戦セットの勝敗を決める。平均石差が正なら勝ち、0なら引き分け、負なら負けである。

```text
勝率 = (勝ち数 + 0.5 × 引き分け数) / 対戦セット数
```

したがって、1組合せ当たりの50対戦セットは100実対局に相当する。

## 1手当たり平均石損

α系列の実戦着手について、同じ局面におけるα=0.0、すなわちEgaroucid level 21相当の最善手評価値との差を測定した。XOTとして事前に与えた8手は測定対象に含めていない。

```text
1手の推定石損 = level 21による合法手の最大評価値 - 実際に選んだ手のlevel 21評価値
```

| α | 測定できた着手数 | 測定できなかった着手数 | 1手当たり平均推定石損 |
| ---: | ---: | ---: | ---: |
| 0.0 | 42,427 | 0 | 0.0000 |
| 0.2 | 42,403 | 0 | 0.0039 |
| 0.4 | 42,276 | 0 | 0.0512 |
| 0.6 | 42,050 | 1 | 0.2219 |
| 0.8 | 41,411 | 0 | 0.6990 |
| 1.0 | 38,373 | 2 | 2.0298 |

測定できなかった3着手では、Egaroucidの`hint`出力に実際の合法手が含まれていなかった。勝敗と石差には影響せず、平均石損の分母からだけ除外した。対戦相手別の値と欠測数は`strength_move_stone_loss_matrix.tsv`および`strength_move_stone_loss_missing_matrix.tsv`に保存した。

## 主な結果

- α=0.0が全体勝率94.38%、平均石差+19.125で最も強かった。
- 混合方策ではα=0.2が最も強く、全体勝率73.38%、平均石差+15.778だった。対Egaroucid level 13は59%、level 15は43%であり、この実験集団内ではlevel 13から15の間に位置した。
- α=0.4は対level 11で60%、対level 13で36%だった。
- α=0.6は対level 9で52%、対level 11で21%だった。
- α=0.8は対level 3で85%、対level 5で34%だった。
- α=1.0は対level 1で33%、対ランダム打ちで100%だった。Policy Network単体は人間の着手らしさを学習しているが、強さを目的としたlevel 21方策とは大きな差がある。

## 実行時間と資源使用量

最初の本実行開始から完了までの経過時間は17時間45分36秒だった。この時間には、安全なプロセス数を確認するための中断、コード修正、再開待ちを含む。最後の連続実行区間は13時間9分17秒だった。

最後の連続実行区間で2秒ごとに測定した値を次に示す。メモリは対戦プログラム単独の使用量ではなく、PC全体の使用量である。

- `strength_games.jsonl`には6,800対戦セットが存在する。
- 対戦IDは0から6,799までで、重複と欠落はない。
